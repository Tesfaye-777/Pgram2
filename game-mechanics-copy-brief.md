# 修仙江湖人生模拟器：机制与文案整理稿

这份文件用于发给 GPT 或其他 AI，帮助其理解当前项目的玩法结构，并进一步优化判定机制、游戏机制和文案。  
项目定位是“命盘 + 词条 + 掷骰判定 + 人生分支”的修仙江湖人生模拟器，不是传统算命报告，也不是纯问答小游戏。

## 一、核心定位

玩家先填写姓名、出生日期、出生时辰、乾造/坤造等信息，生成一份“入世命局”。随后玩家经历二十重人生关卡，每一关阅读场景、选择行动、触发命格判定，并根据判定结果改变五维属性、人生状态和最终结局。

整体体验目标：

- 像一卷古风命册，而不是现代数据面板。
- 像一次江湖人生推演，而不是简单算命报告。
- 每个选择都要有明确后果，失败也继续推进故事。
- 文案要有“称骨批命、江湖评断、一语定局”的气质，但不能空泛、不能机械套模板。
- 所有内容都应围绕玩家的一生展开，少写“系统感”“参数感”“这是娱乐内容”等出戏表达。娱乐声明已单独显示，不需要在剧情里反复强调。

## 二、玩家生成机制

玩家输入：

- 姓名
- 出生日期
- 历法类型：新历/农历，目前只记录类型，尚未接真实换算
- 出生时辰：子、丑、寅、卯、辰、巳、午、未、申、酉、戌、亥
- 性别：乾造/坤造

当前第一版使用本地 mock 规则生成命局。生成逻辑是稳定的：同名、同日期、同时辰、同乾坤造会得到同一组本地娱乐命局。

需要保留的结构：

- 后续可以替换为真实后端 API、AI Skill 或 Agent Skill。
- 当前本地规则只负责生成 seed、词条、五维属性、初始评语。

## 三、五维属性

五维属性范围为 0-100，是角色命盘的核心能力底数。

1. 福缘
代表运气、贵人、意外转机、逢凶化吉。
主要影响：随机事件、奇遇、贵人、危机翻盘、生还概率。

2. 财势
代表资源、财富判断、商业嗅觉和资产增长能力。
主要影响：投资、创业、经商、财富机会、资源调度。

3. 心性
代表稳定、忍耐、自控、抗压和长期选择能力。
主要影响：冲动风险、感情抉择、长期修行、危机中是否失控。

4. 魄力
代表胆识、行动力、冒险精神和破局能力。
主要影响：高风险选择、创业、对抗、关键时刻主动出手。

5. 悟性
代表理解力、学习力、洞察力和看穿局势的能力。
主要影响：学业、谋略、识人、破解困局、发现隐藏选项。

游戏内原则文案：

> 五维决定能力，人生状态决定处境，词条决定命运偏差，骰子决定人生无常。

## 四、人生状态

人生状态不是命盘天赋，而是玩家在流程中累积出来的处境。

1. 声望
代表名声、外界评价、社会影响力。
高声望带来信任、机会和公开支持；低声望容易被质疑、孤立或阻碍。

2. 资产
代表财富、资源、现实筹码。
影响创业、投资、迁徙、疗伤、危机处理和晚年结局。

3. 关系
代表人脉、亲友、贵人、盟友和情感连接。
影响求助、合作、感情线、贵人线、家族/门派路线。

4. 压力
代表精神、现实和命运压力。
压力越高，判定越难，并可能放大失败代价。

当前压力规则：

- 初始压力为 0。
- 压力惩罚 = floor(压力 / 20)。
- 压力惩罚会并入“状态加值”显示。
- 成功时，部分稳妥选择、修养选择、提升心性的选择会降低压力。
- 失败和大失败会增加压力，但目前已做过一次削弱，避免压力过快满值。

## 七、选项结构

每个选项可包含：

- id：选项唯一标识
- label：选项标题
- description：选项简述
- requiredAttribute：关键五维属性
- difficulty：原始难度
- riskLevel：风险层级
- relatedStatus：关联人生状态
- effects：基础五维变化
- stateEffects：基础人生状态变化
- requirement：解锁条件
- traitInteractions：词条交互
- routeEffect：路线变化提示
- resultTexts：自定义五档结果文案

风险层级：

- safe：稳妥
- normal：正途
- risky：冒险
- fate：逆命
- heaven：天命

旧版 risk 字段仍保留：

- low
- medium
- high

## 八、判定机制

当前判定函数：

`resolveChoiceCheck(save, choice)`

输入：

- 当前命局
- 当前五维属性
- 当前人生状态
- 当前场景
- 当前选项
- 当前词条

输出：

- 骰点
- 关键属性
- 属性底数
- 原始难度
- 折算后所需点数
- 命盘词条加值
- 状态加值
- 压力惩罚
- 临场修正
- 最终总点数
- 判定等级
- 实际五维变化
- 实际人生状态变化
- 触发的词条效果
- 路线效果
- 结果文案

### 当前核心公式

现在界面展示的是“折算后所需点数”，而不是直接拿 80、100 这种大数让骰子去追。

原始难度先与关键五维属性底数相减，再折算：

```text
差值 = max(0, 原始难度 - 属性底数)
折算后所需点数 = ceil(差值 / 8) + 风险偏移
```

风险偏移：

- 稳妥：+0
- 正途：+1
- 冒险：+2
- 逆命：+3
- 天命：+4

折算后所需点数限制在 2-18 之间。

最终总点数：

```text
最终总点数 =
骰点
+ 命盘词条加值
+ 状态加值
- 压力惩罚
+ 临场修正
```

界面上“状态加值”实际应理解为：

```text
状态加值 = 声望/资产/关系加值 - 压力惩罚 + 临场修正
```

## 九、骰子机制

当前骰子是六面骰，只能掷出 1-6。

骰子展示：

- 六面体
- 朝上面显示汉字数字：壹、贰、叁、肆、伍、陆
- 点击选项后进入判定界面，骰子旋转，逐渐停下

骰点生成：

- 当前不是完全随机，而是基于 seed、关卡 index、已选记录数量、choice id 做 deterministic roll。
- 同一命局、同一流程、同一选择会得到稳定骰点。
- 这样可以保持“命盘推演”的稳定性。

特殊骰点规则：

- 如果骰点为 6，并且最终判定成功，直接升级为大成功。
- 如果骰点为 1，并且最终判定失败，直接降为大失败。

## 十、判定等级

当前五档结果：

1. 大成功
条件：最终总点数 >= 所需点数 + 15，或骰点为 6 且原本已成功。
效果：获得额外奖励，可能留下更强路线伏笔。

2. 成功
条件：最终总点数 >= 所需点数。
效果：获得选项原本收益。

3. 不尽人意
条件：最终总点数低于所需点数 1-10。
效果：获得部分收益，但附带压力或资源代价。

4. 失败
条件：最终总点数低于所需点数 11-25。
效果：收益折损或转为负面结果。

5. 大失败
条件：最终总点数低于所需点数 25 以上，或骰点为 1 且原本失败。
效果：触发严重后果，压力、资产、关系、声望等受到较大影响。

## 十一、词条判定交互

词条参与判定的方式：

1. 如果词条对当前关键五维有 statBonus，则折算为少量判定加值。
2. 如果词条 tag 与当前场景/选项语义匹配，则根据稀有度增加判定加值。
3. 如果词条是凶签，且 tag 匹配当前场景，可能扣分。
4. 如果选项设置了 traitInteractions，匹配后可增加 checkBonus 或 failureGrace。
5. 特定词条有特定场景修正：
   - 逢凶化吉在“命运低谷/三船沉江”类场景中增强失败修正。
   - 逆天改命在最终关增强失败修正。
   - 大器晚成在成长类后期场景中有额外加值。
   - 心火易燃在冒险路线中可能有一点冲劲，但失败修正变差。

## 十二、隐藏选项机制

隐藏选项来源：

1. 特定场景匹配特定词条。
2. 当前关键五维达到阈值。

当前隐藏选项阈值：

- 第 1-5 关：关键五维 >= 68
- 第 6-10 关：关键五维 >= 72
- 第 11-15 关：关键五维 >= 76
- 第 16-20 关：关键五维 >= 80

隐藏路线的文案方向：

- 不要写“真正发力的是某属性”。
- 不要像系统解释属性来源。
- 应写成“暗线、伏笔、局势、破绽、借势”等具体剧情推进。

当前隐藏路线结果文案模板：

```text
大成功：
【场景】你走出「选项」。描述。命骰六点，暗线反成明局。你把旁人忽略的缺口握在手里，眼前之事顺势打开，还替日后留下一处可借的人情。

成功：
【场景】你走出「选项」。描述。这条暗线走通了。你没有声张，只把关键一步落在旁人看不见的地方；等众人回过神时，局面已经偏向你这边。

不尽人意：
【场景】你走出「选项」。描述。暗线只走通半截。你摸到一点转机，却没能完全压住局面，后续还要再补一手。

失败：
【场景】你走出「选项」。描述。这条暗线没能接上明局。你临场变招被人看出犹豫，反倒多添一处破绽。

大失败：
【场景】你走出「选项」。描述。命骰一点，暗线反噬。你想藏住的破绽被推到台前，后续处境也因此受牵连。
```

## 十三、普通选项结果文案

当前普通选项已有一批定制后续文案，放在 `lib/choiceOutcomeNotes.ts`。

基本原则：

- 每个普通选项至少有成功、失败两段后续。
- 大成功会在成功文案后追加一句“此局另有余势……”之类的补充。
- 大失败会在失败文案后追加一句更严重的牵连。

需要继续优化的方向：

- 不要写空泛句，例如“这一步落得稳，所得与代价都在你能承受的范围内”。
- 不要写机械句，例如“真正发力的是心性”。
- 不要写“你选择了某某，此路属某某之选”这种系统式解释。
- 要围绕当前事件本身给出具体结果：
  - 谁知道了？
  - 谁误会了？
  - 损失了什么？
  - 得到了什么？
  - 后续局势怎样变化？
  - 哪条关系、资产、名声或压力发生了变化？

推荐文案结构：

```text
【关卡名】你选择「选项名」。
第一句：直接写选择实施后的具体场面。
第二句：写结果对人物关系、钱财、名声或处境的影响。
第三句：写这件事如何成为后续伏笔。
```

- 修仙江湖
- 命册批语
- 古风但不要硬拗文言
- 有画面感
- 有现实处境
- 有因果
- 少废话

适合使用的词：

- 命册
- 灵签
- 命盘
- 入世
- 关口
- 暗线
- 伏笔
- 破局
- 借势
- 劫数
- 局面
- 余地
- 人情
- 名声
- 财路
- 根基
- 回响

需要避免的词和句式：

- “这是娱乐内容”

- “参数”
- “系统判定”

## 十五、结局机制

最终结局综合：

- 最终五维平均值
- 声望
- 资产
- 关系
- 压力
- 词条稀有度分
- 大成功次数
- 大失败次数
- 关键选择

当前评分结构：

```text
总分 =
五维平均值
+ 声望 * 0.16
+ 资产 * 0.16
+ 关系 * 0.14
- 压力 * 0.16
+ 词条稀有度总分 * 0.9
+ 大成功次数 * 2
- 大失败次数 * 3
```

当前结局档位：

- SSS：上上格
- S：上格
- A：中上格
- B：中平格
- C：下格

结局文案需要更像命书总批，但要具体：

- 写清楚财、名、人、压、运的实际结果。
- 写出玩家一生最关键的选择。
- 写出主签如何影响此生。
- 写出哪里走旺，哪里有亏欠。
- 不要写空泛鸡汤。
